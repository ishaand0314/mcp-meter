The tax nobody sees

Every MCP server your agent connects to registers a set of tools, and each tool's name, description, and input schema gets stuffed into the model's context on every single turn, before you've written a word of your prompt. Connect a handful of servers and that overhead compounds into thousands of standing tokens and real monthly spend, with no visibility into which server or tool is responsible. MCP Meter reads your actual configured servers and shows you the breakdown in seconds, no prompts required.
